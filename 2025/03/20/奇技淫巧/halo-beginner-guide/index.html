0. 前言（牢骚）

一直有着想要建一个自己的 blog 的想法，一个是可以记录自己的学习历程，另一个是可以分享自己的文章，还有就是感觉很酷。其实我总体还是挺喜欢分享自己的文章，帮助他人的，不知道是出于什么心理。

因此，我决定记录下我建站的过程，希望我遇到问题、解决问题的过程记录能在他人建立属于自己的 blog 时提供一定的帮助。

1. Blog 框架选择

建站前，首先要先选择要通过什么框架建站。一开始我什么都不知道，以为 Github Pages 就够了，结果一问说是静态网站，没有后端。这个虽然可以用动态穿透解决，但是总体还是不太好。

下面是一些常用 blog 框架的调研：

嗯。综合上面的优缺点，我最终选择 Halo 作为我第一个 blog 的框架。因为我希望有后端，那就最好选择 Typecho、Halo 或者 WordPress，但是 Typecho 停止更新了，而 WordPress 又太臃肿了（而且高级版要钱！），这样看来最好的选择似乎就是 Halo 了。

附 Halo 官网、Halo Github Repo、Halo 文档、Halo 社区地址。

2. 服务器搭建

既然选择 Halo 作为博客框架，那么就需要有一个支持 Java 运行环境的服务器作为后端。服务器一般有云服务器和自己搭一个 NAS 两种选择。自己搭一个 NAS 感觉比较麻烦，而且宿舍内晚上会断网，不太方便。所以我选择使用云服务器。

2.1. 云服务器厂商选择

提示

截止到我写作的时候（2024.11.19），阿里云“云工开物”计划对大学生有 300 元的补贴，而 Azure 则有 100$ 的补贴，同时还额外提供免费的 1 核 1 G 的 Windows 和 Linux 的服务器各一个 (不计算在 100$ 的消耗中)。如果你是大学生，可以趁机薅一把羊毛。具体认证过程可以自己百度搜一下。(申请的时候没有记录，Sorry) 但是要注意，Azure 免费的 1 核 1 G 的云服务器可能不够 Halo 用的哈。

总体来说，国内阿里云、腾讯云会比国外服务器更加便宜。但如果使用国内服务器，默认 80 和 443 端口是关闭的，你需要备案之后才能使用。备案过程快速与否比较看运气，并且有很多限制和审查。如果你准备备案的话，出于阿里云 300 元补贴的考虑，我建议你选阿里云的服务器，一年大概 100 元。当然续订可能会更贵，到时候你直接备份数据然后重新订一年新的就行。如果你不想备案，我建议用腾讯云。Azure 虽然给了 1 核 1 G 的云服务器，但是 1G 的内存对于 Halo 来说还是太小了。而如果你想买性能更强的服务器，100 $ 又不够用。我是纯懒狗，所以我选择买腾讯云的境外服务器。

但是还是给出阿里云 ICP 备案流程。必须强调，网络不是法外之地。作为国内的公民，我们应该理解并尊崇国家政策。

以下以腾讯云为例，说明整个建站流程。

4. 域名购买

域名的提供商有很多，国内有阿里云、腾讯云等，国外也有 GoDaddy、Namecheap、Google Domains 等，随便选一个渠道购买即可。这里就以阿里云为例。

4.1. 域名注册

进入阿里云域名注册界面购买域名。这里有几个点要注意：

  1. 购买域名的时候不要只看首年价格，首年一般都有优惠，一定要看看续费多少钱。
  2. 一般一次购买多年会有优惠。
  3. 不同域名后缀除了名字之外没什么区别。.com 一般比较贵，因为是最广泛认可和使用的域名后缀，用户记忆度也大多较好。对于博客网站，.top、.blog 等都可以，完全可以挑比较便宜同时也不生僻的后缀购买。(注：阿里云没法购买 .blog 后缀的域名，像 .moe 这样的域名后缀阿里云也缺失。如果真的想买可以去国外域名提供商看看，比如 NameSilo、GoDaddy 等)

4.2. 域名解析

购买完域名后，进入域名控制台，点击刚刚购买的域名右边的“解析”。

5. 配置反代和 HTTPS

可以看到，配置了域名之后还是需要指定 8090 端口，而网站一般都是使用默认的 80 端口或 443 端口，这个时候就需要配置反向代理了。

什么是反向代理

反向代理（Reverse Proxy）是一种服务器配置类型，它位于用户客户端与目标服务器之间，充当用户请求的中介。简单来说，反向代理服务器接受客户端的请求，然后将这些请求转发给后端的一台或多台服务器，并将从后端服务器接收到的响应返回给客户端。

之前，我们已经在部署应用时部署了 Nginx Proxy Manager，接下来只要通过 81 端口 http://你的域名:81 进入其控制台即可。默认账号是 admin@example.com，密码是 changeme，首次进入需要编辑用户和修改密码。

然后进入 Dashboard 部分，点击 Proxy Hosts，再点击 Add Proxy Host。

然后按照 Halo 官方配置反向代理教程设置即可。如果有兴致，也可以举一反三将 NPM 也用一个 (子) 域名来反向代理。

设置完后，你可以通过 https://你的域名、http://你的域名/console 直接访问你的博客和博客后台了。
