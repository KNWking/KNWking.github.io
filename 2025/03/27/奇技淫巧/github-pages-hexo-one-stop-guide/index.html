默认的 Hexo 主题也挺不错，但是我个人有更喜欢的主题，所以我准备换成前面提到过的 redefine 主题。

打开 redefine GitHub repo，然后根据里面的说明来。指引文件说“如果你的 Hexo 版本在 5.0 及以上，推荐通过 npm 安装”，那我们就首先看一下 Hexo 版本

我这里显示

说明我的 Hexo 版本是 7.3.0 > 5.0，因此按它说的采用 npm 安装，首先在你的 blog 文件夹打开 git bash，然后输入

安装完成后，在 Hexo 配置文件 _config.yml 中将 theme 设置为 Redefine。位置在之前最后更改的 deploy 的上面，当然你也可以用 Ctrl + F 搜索得到。

然后我们在你的 blog 文件夹 (Redefine Docs 中的“Hexo 根目录”就是你的 blog 文件夹) 中新建一个 _config.redefine.yml 文件 (新建文本文档再改名字就行)，然后把这里的所有内容复制进去并保存。

创建这个文件的目的是为了方便你在升级主题时，不会丢失你的配置。所以以后所有改动都要在 _config.redefine.yml 文件中进行，不建议直接修改主题文件，因为更新主题时会覆盖你的改动。
